# 本地启动与服务端口

## 固定端口

| 服务 | 端口 | 命令 |
| --- | --- | --- |
| Web | `3100` | `npm run dev:web` |
| API | `4100` | `npm run dev:api` |
| Worker | 无固定 HTTP 端口 | `npm run dev:worker` |

前端默认会把本地 API 指向 `http://127.0.0.1:4100`。相关逻辑在 `apps/web/lib/api.ts`。

## 常用命令

```powershell
npm run dev:web
npm run dev:api
npm run dev:worker
npm run typecheck
npm run build
npm run docs:maintenance
```

`apps/web` 和 `apps/api` 的 dev 脚本会跑 `infra/scripts/dev-preflight.mjs`，用于清理 Windows 上常见的端口占用和 Prisma 临时文件残留。

## 健康检查

API：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:4100/health | Select-Object -ExpandProperty Content
```

Web：

```powershell
Invoke-WebRequest -UseBasicParsing http://127.0.0.1:3100/generate | Select-Object -ExpandProperty StatusCode
```

## 本地数据和环境

本地常见组合是 JSON store、inline queue、本地或兼容 provider。真正生产环境需要补齐真实 provider、对象存储、支付、Redis、SMTP 等配置。

不要在维护文档里写真实密钥。涉及 `.env`、token、证书、CI/CD 的改动必须先确认，不要顺手改。

## 自动地图更新

改了 API 路由、页面、包结构或环境变量引用后，运行：

```powershell
npm run docs:maintenance
```

再检查 `docs/maintenance/generated` 的 diff，确认地图和实际代码一致。

# 管理后台与安全审核

## 管理后台入口

页面：`apps/web/app/admin/page.tsx`

主要能力：

- 运营指标。
- 用户列表和详情。
- 生成任务列表。
- 图片管理。
- 订单和套餐管理。
- 安全规则。
- 安全事件人工复核。
- 审计日志。
- 运维告警和事件。

## 安全规则

安全规则用于拦截或复核 prompt、上传图片、生成结果。维护时要区分：

- `BLOCK`：直接拦截。
- `REVIEW`：进入人工复核。
- `PASSED`：允许继续。

安全判断不要只做前端校验，后端和 worker 侧必须兜住。

## 安全事件和申诉

生成页会在内容被拦截或需要复核时提供申诉入口。后台处理后会写安全事件状态和审计日志。

维护重点：

- 复核动作必须记录 admin、目标对象、原因。
- 不要让安全事件状态和任务状态分叉。
- 被拦截任务的积分处理要和生成失败补偿一致地检查。

## 运维指标

后台指标来自 API 聚合，不是前端自算。出现指标不准时，先查 API 数据源和 store，再查展示层。

# 图片存储与预览流程

## 数据流

```text
worker 得到图片数据
-> packages/storage 保存原图和缩略图
-> 数据库记录 GeneratedImage
-> history/favorites/images 页面读取列表
-> preview/download 接口生成可访问 URL
```

## 关键对象

- `GeneratedImage`：图片记录，包含 taskId、userId、thumbnailUrl、publicUrl、尺寸、可见性、删除状态。
- `ReferenceImage`：参考图上传记录。
- storage key：真实文件存储位置，不应直接暴露为永久公开地址。

## 预览和下载

前端预览组件在 `apps/web/components/GeneratedImagePreview.tsx`。

维护重点：

- 列表优先用 thumbnail。
- 大图预览保持原始比例。
- 下载走签名 URL 或受控接口。
- 私有图片不能被其他用户访问。
- 删除通常是软删除，注意列表过滤。

## 可见性和收藏

历史页、收藏页、详情页共享图片状态。改收藏或删除时要验证：

- history 列表变化。
- favorites 列表变化。
- detail 页面权限。
- admin 图片管理是否仍能看到正确状态。

# 管理页路由恢复加载设计

## 目标

修复从其他页面返回 `/admin` 时数据不重新加载、必须手动刷新才能恢复的问题，并避免单个后台接口失败拖垮整页数据。

## 根因

- 管理页仅在授权状态或筛选条件变化时调用 `load()`，没有把重新进入 `/admin` 作为加载信号。
- `load()` 使用 `Promise.all` 聚合多个接口，任一接口失败后所有成功结果都不会落入页面状态。

## 方案

- 使用 `usePathname()` 识别管理页重新成为当前路由，并触发一次完整加载。
- 将完整加载改为 `Promise.allSettled`，各模块成功结果独立更新；失败模块保留旧数据并汇总提示。
- 保留现有请求序号控制，防止旧请求覆盖新筛选结果。
- 增加静态回归契约，锁定路由恢复加载和部分失败容错。

## 验收

- 从其他页面返回管理页会重新请求后台数据。
- 单个后台接口失败时，其余模块仍显示成功返回的数据。
- 定向测试、类型检查、全量测试和构建通过。

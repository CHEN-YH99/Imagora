# Imagora 第一波实现计划（独立完成 + 外部依赖占位）

目标：把项目从"沙箱演示"推到"技术上可上线"。本波**不依赖任何外部密钥**即可全部完成；
凡是需要真实密钥/账号的（AI、支付、审核、邮件），只写**接口骨架 + 工厂 case + .env.example 占位**，
后面你给钥匙就能直接接通，不返工。

---

## A. 本波直接实现（不依赖外部密钥）

### A1. 会话安全收敛（P0，改动集中）
现状：后端已发 HttpOnly Cookie，前端 `apps/web/lib/api.ts` 还在用 `localStorage` 存 token，双轨。
- `apps/web/lib/api.ts`：删除 `getStoredToken`/`setStoredToken` 的 localStorage 读写，统一依赖 Cookie（fetch 已带 `credentials:"include"`）。
- 各前端页面（account/admin/generate/history/favorites/orders/login/register/layout/AppFrame）：移除手动传 token 的逻辑，改为靠 Cookie 自动携带。
- `apps/api/src/main.ts`：登录/注册响应体中的 `token` 字段保留兼容，但前端不再持久化；上线开关 `AUTH_BEARER_COMPAT` 控制是否还接受 Bearer（默认开，留迁移窗口）。
- CSRF：确认 `setSessionCookie` 的 `SameSite=Lax`，写操作依赖同源 + 自定义 header；如不足再加 CSRF token。

### A2. 密码重置全链路（P1，独立好闭环）
- **数据模型**：新增 `passwordResetTokens` 表（id、userId、tokenHash、expiresAt、usedAt、createdAt）。
  - `packages/database/prisma/schema.prisma` 加 model + 索引。
  - `packages/shared/src/index.ts` 加类型，`StoreData` 加字段。
  - `packages/database/src/index.ts`：JsonStore（seed/normalize）+ PrismaStore（read/write）双实现都要补。
- **API**（`apps/api/src/main.ts`）：
  - `POST /api/auth/password-reset/request`：按邮箱发重置链接；**不泄露邮箱是否存在**（永远返回成功）；token 只存 hash，TTL 走 `PASSWORD_RESET_TOKEN_TTL_MINUTES`（默认 30）。
  - `POST /api/auth/password-reset/confirm`：校验 token hash + 未过期 + 未使用 → 改密码 → 标记 used → 失效该用户所有 session。
  - 两个接口都加 rate limit rule。
- **邮件**：新建 `packages/mailer`，定义 `Mailer` 接口 + `ConsoleMailer`（开发期把重置链接打到日志/控制台）+ SMTP/阿里云邮件骨架占位（见 B4）。
- **前端**：新增 `/forgot-password`、`/reset-password` 页面，登录页加"忘记密码"入口。

### A3. lint / format 门禁（P0，文档吹了但没装）
- 装 ESLint + Prettier + typescript-eslint，根目录配置（flat config）。
- 根 `package.json` 加 `lint`、`lint:fix`、`format` 脚本。
- 不强求一次清零历史告警：先跑通，明显问题修掉，其余记 TODO。

### A4. 后台用户详情 + 查询补全（P1）
- **API**：
  - 新增 `GET /api/admin/users/:userId`：返回用户资料 + 积分账户 + 近期积分流水 + 近期任务 + 近期订单。
  - 后台列表（users/tasks/images/orders）查询参数补：`createdFrom`/`createdTo` 时间范围；订单列表补 `orderNo` 搜索。
- **前端**（`apps/web/app/admin/page.tsx`）：用户详情抽屉/区块 + 时间范围与订单号筛选控件。

### A5. 缩略图真实生成（sharp，npm 包不需密钥）
- `apps/worker` 装 `sharp`。
- `apps/worker/src/main.ts` 的 `createImage`：位图（png/jpeg/webp）用 sharp 真 resize 出小尺寸缩略图（如最长边 512），SVG 保持现有 viewBox 缩放逻辑。
- 失败兜底：sharp 出错时退回原图复用，不阻断生成。

## B. 外部依赖占位骨架（写好接口，等你给密钥）

每个都做成：`Provider 接口实现 + 工厂 switch case + .env.example 占位 + 未配置时抛清晰错误`。
默认走 mock，绝不影响本地演示。

### B1. 国内 AI provider（通义万相 / 即梦）
- `packages/ai-providers/src/index.ts` 加 `TongyiImageGenerationProvider` 骨架，工厂加 `case "tongyi"`。
- 真实调用代码留 TODO + 占位，`.env.example` 加 `DASHSCOPE_API_KEY` 等。

### B2. 国内支付（微信支付 / 支付宝）
- `packages/payments/src/index.ts` 加 `WechatPaymentProvider`、`AlipayPaymentProvider` 骨架（createPayment + verifyWebhook 验签留 TODO）。
- 工厂加 `case "wechat"` / `case "alipay"`；`.env.example` 加商户号/密钥/证书路径占位。
- webhook 路由已支持 `:provider` 参数，无需改路由。

### B3. 内容审核第三方（阿里云内容安全 / 腾讯天御）
- `packages/safety/src/index.ts` 加 `AliyunSafetyProvider` 骨架（checkText/checkImage），保留本地规则作为第一层。
- 工厂支持组合：本地规则 + 第三方；`.env.example` 加审核密钥占位。

### B4. 邮件服务（密码重置发信用）
- `packages/mailer`：`ConsoleMailer`（默认）+ `SmtpMailer`/阿里云邮件骨架占位。
- `.env.example` 加 SMTP / 邮件推送占位。

### B5. 对象存储（国内 OSS/COS）
- 好消息：现有 `S3CompatibleObjectStorage` 兼容阿里 OSS/腾讯 COS，**基本只改配置**。
- `.env.example` 补 OSS/COS 的 endpoint/region/bucket 占位说明，不改代码逻辑。

## C. 验收
- `npm run typecheck && npm test && npm run build` 全绿。
- 新增 lint 跑通。
- 密码重置：请求→控制台拿链接→确认改密→旧 session 失效，端到端手测 + 加集成测试。
- 会话收敛后：登录/生成/历史/支付/后台全流程在 Cookie-only 下可用。
- 缩略图：位图任务产出的缩略图尺寸明显小于原图。
- 占位 provider：切到 mock 不受影响；切到国内 provider 未配密钥时报清晰错误。

## D. 不在本波（等你/后续）
- 真实 AI/支付/审核/邮件联调（等密钥）。
- 生产可观测性（结构化日志落盘、Prometheus、告警通道）。
- 前端 Playwright E2E、并发压测、CI/CD。
- P2：图生图/变体/放大/局部重绘/批量。
- 合规框架（ICP/算法备案/商户资质）——你说先留着。
